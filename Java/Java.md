### 형식

### 객체 생성과 사용

`new 연산자 사용`

### 접근 제어자

| private       | 클래스 안에서만 접근 가능                 |
| ------------- | ------------------------------ |
| **public**    | **전체에서 접근 가능**                 |

| **friendly**  | **기본은 public**                 |

### 오버로딩

- 중복되는 메소드명을 클래스 안에서 정의 할수 있도록 하는 것
- 같은 함수명으로 다양한 값을 받아 처리
- 함수명 하나만 기억하면 된다  

### 생성자

- 객체가 생성될 떄 한 번 호출되는 메소드
- 자동으로 호출되는 메소드
- 리턴형이 없고 클래스명과 함수명이 같다
- public

### this, this()

- 객체  자신을 참조하는 참조형 클래스변수 또는 키워드
- `this.멤버변수` 
- `this.메소드()`

### static

- 접근 제어자 static 테이터형 멤버변수명;
- 객체가 생성되기 전에 멤버변수와 메소드 사용 가능
- 멤버변수는 생성되는 모든 객체에서 전역변수로 사용

### package

- 자주 사용되는 클래스를 모아 놓은 폴더
- 패키지명은 소문자로 사용한다(관례)

### import

- C언어의 #include와 유사함.
- 클래스를 포함하고자 할 경우



## 7. 상속(inheritance)

- 객체와 객체를 연관시키고 계층구조를 만들어 코드를 재사용.
  (상위 클래스의 코드를 그대로 사용 가능)
- 상속시켜주는 클래스 - 부모클래스, 상위클래스, parent class, spuer class
- 상속 받는 클래스 - 자식 클래스, 하위클래스, chils class, sub class
- 오직 `한 클래스만을 상속`할 수 있다.
- 최상위 클래스 - Object
- 상위 클래스의 접근지정자에 따라 접근이 제한되어있다.
- `extends` 키워드 사용


### 멤버 상속

- 상위 클래스에 지정된 `private 멤버`는 해당 클래스에서만 사용할수 있고,
  하위 클래스에서는 사용할 수 없다.
- 상위 클래스에 지정된 `public 멤버`는 해당 클래스 뿐만 아니라 하위 클래수에서도 사용할 수 있으며 외부에서도 언제든지 사용할 수 있다.
- 상위 클래스에 지정된 `protected 멤버`는 같은 패키지 일 때 public 멤버처럼 사용.
  다른 패키지인 경우는 상위 멤버를 하위 클래스 안에서만 사용할 수 있다. 
- 상위 클래스에 `default 멤버`는 같은 패키지 안에 있을 때는 public처럼 사용
  다른 패키지인 경우는 하위 클래스에서 아용 할수 없고, 외부에서도 사용 할 수 없다.

### 오버라이딩

- 무보 클래스의 메소드를 `자식 클래스에 서 다시 재정의` 하는 것
  - 함수명이 같아야 한다.
  - 함수의 매개변수가 같아야 한 다.
  - 리턴형이 같아야 한다.

### final

- 클래스와 메소드에 붙는 경우 - 상속과 오버라이딩 불가
- 변수에 사용되는 경우 - 값을 수정 할 수 없다.

### 상속관계에서 생성자의 호출순서

1. 상위 클래스 생성자에서부터 하위 클래스 생성자 호출 
2. 생성자에 매개변수가 있는 경우 자동호출 안됨 -> `supter` 키워드 사용

### super

- 하위 클래스에서 상위 클래스를 참조하여 접근할 때 사용
- `super.멤버변수명`, `super.메소드명()` 으로 사용 

### 참조변수를 이용한 형 변환과 호출

- 캐스트 연산자 사용 `( )`
- 형 변환의 규칙
  - 하위 클래스에서 상위 클래스로 형변환할 수 있음.
  - 접근 지정자에 따라 접근이 결정된다.
- 상위클래스로 형변환시 하위클래스의 멤버에 접근 할 수 없다.
- 상위클래스의 메소드가 하위 클래스에 오버라이딩 된 경우에
  상위 클래스 참조변수에서 오버라이딩 된 메소드를 호출하면 하위 클래스의 메소드가 실행된다. 



## 8. 추상 클래스(abstract class)

- 하위에서 구현할 메소드의 원형을 가진  상위 클래스

## 9. 인터페이스(interface)

### 추상클래스와 인터페이스

- 공통점
  - 상위 클래스로만 사용 가능
  - (추상메소드)하위에서 구현할 함수의 원형을 선언 
- 차이점
  - 추상클래스 = 일반 맴버 + 추상 메소드
    인터페이스 = 추상 메소드 + 상수
  - 추상 클래스는 추상 메소드를 일부 구현 가능
    인터페이스는 모두 구현해야 함(강제적)
  - 추상 클래스는 다중 상속 가능
  - 추상 클래스는 인터페이스가 인터페이스를 다중 상속 

### 기본 형식

``` java
interface 인터페이스명{
  public static final 데이터형 변수명 = 값;
  public abstract 리턴형 메소드명(매개변수 ..)
}

// 위의 키워드를 생략해서 사용 할 수 있다.
interface 인터페이스명{
  데이터형 변수명 = 값;
  리턴형 메소드명(매개변수 ..)
}
```

- `implements`  키워드 사용
- 인터페이스명 관례 : ~able



## 10. 예외처리

### 개념

- 예외 처리는 프로그램 실행 도중에 발생되는 오류를 처리하기 위한 부분.
- 에러 : 수정할 수 없는 것
- 오류 : 수정 가능, 알려주는 역할
- 키워드 : `try`, `catch`, `finally`, `throws`

### 형식

``` java
try {
    // 예외가 발생할 수 있는 문장
} catch (Exception e){
    // 예외가 발생핼을 때 처리하는 문장1
} catch (Exception e){
    // 예외가 발생핼을 때 처리하는 문장2
} finally {
    // 예외처리와 상관헙이 항상 실행되는 문장
}
  
```

throws : 모든 예외 처리를 try ~ catch 안의 괄호에 넣어 처리하기 힘들 때, JVM 또는 예외처리 클래스를 지정.



## 11. 내부 클래스와 익명 클래스

### 내부 클래스(inner class)

- 클래스 안에 클래스를 정의 한 것
- 내부 클래스는 외부 클래스의 모든 자원 사용 가능
- new로 생성
- 외부 클래스를 통해 내무 클래스 사용
- 선언에 따라 인스턴스 내부 클래스, 정적 내부 클래스, 지역 내부 크래스로 나눌 수 있다.

### 익명 클래스(anonymous class)

- 클래스명이 없다
  - 일맨 내무 클래스 : 외부클래스명$내부클래스명.class로 컴파일됨.
    익명클래스 : 외부클래스명$1.class 파일로 컴파일됨.
- 선언과 생성이 동시에 이뤄진다.
- 일회성



## 12. 유용한 클래스

### wrapper class

- 기본 데이터형을 객체로 다룰 수 있도록 해준다.
- 기본 데이터형 + 메소드
- ex) `boolean` -> `boolean`, `int` -> `Integer` ..
- autoboxing
  - wrapper 객체와 값 사이의 변환을 자동 처리해주는것.
  - 기존 wrapper 클래스의 메소드를 간략하게 사용할 수 있게 해준다.
